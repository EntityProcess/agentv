# Solutions

`docs/solutions/` is the repository knowledge base for documented fixes,
decisions, conventions, and best practices. Category folders such as
`best-practices/`, `runtime-errors/`, `tooling-decisions/`, and
`architecture-patterns/` hold the pages.

Each solution document should include YAML frontmatter with:

- `title`
- `module`
- `date`
- `problem_type`
- `component`
- `severity`
- optional `tags`
- optional `applies_when`

Keep files focused on reusable lessons: what happened, what decision or fix was
chosen, why it works, when to apply it, and any related solution links. Do not
store private tracker state, local branch names, generated screenshots, or raw
session transcripts here.
